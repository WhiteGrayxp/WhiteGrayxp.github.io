深度强化学习在车联网信道接入中的应用

· 2020-05-18 · # 车辆网 # DRL # 论文阅读

🚕这篇文章主要将近年来的几篇使用DRL设计车联网信道接入方案设计的文章进行总结整理。

1. Deep Reinforcement Learning Based Mode Selection and Resource Allocation for Cellular V2X Communications

这篇文章研究的是Cellular V2X通信中的传输模式选择与资源分配联合优化问题，文章提出了基于深度强化学习的分布式算法，在满足V2V用户对时延和可靠性需求条件下，最大化V2I用户的总传输速率。考虑到本地模型训练以及新加入用户训练的约束（训练数据少、算力不足、数据隐私等），文章进一步提出了包含两个时间尺度的联邦DRL算法（Federated DRL)以得到稳定鲁棒的模型，具体而言，首先在大尺度时间上使用图理论将用户基于信道相似性进行分簇（谱聚类），然后在小尺度时间上使用联邦学习得到簇内的全局模型。新加入的用户可以直接基于相应簇内训练好的模型进行接入操作。

文章研究的场景图如下所示：

将车辆分为使用V2I通信模式的I-VUE，以及使用V2V通信的V2V-Pair，即文章考虑将每一个V2V通信的收发机看作一个配对，然后作为智能体进行决策。文章考虑LOS和NLOS通信环境并存，即部分V2V-Pair之间可能被建筑或其它车辆阻挡，这部分V2V-Pair需要通过基站中继来完成通信，即使用V2I通信模式的V2V-Pair。文章的Problem formulation此处略过，简单概括就是在满足I-VUE最低传输速率、V2V-Pair的时延和可靠性（中断概率）、以及为了简化问题设定的每个V2V-Pair只能分配一个资源块、不同V2V-Pair可以接入同一资源块、每个资源块只能分配给一个使用V2I通信模式的V2V-Pair，以及最高发射功率等等约束条件下，希望最大化I-VUE的速率。这是一个混合非线性整数优化问题，很难求解，因此文章提出使用DRL来进行优化。

2. Deep Reinforcement Learning Based Resource Allocation for V2V Communications

这篇文章是作者对两篇会议论文工作的总结，将多播Multicast和单播Unicast两个场景整合，（两篇会议得到一篇期刊😅）

文章假设V2I通信和V2V通信共享频谱资源，优化目标都是使用DRL寻找满足V2V通信时延同时最小化对于V2I通信干扰的最优策略，将每个V2V link看做一个agent，在不需要全局信息的情况下，依靠自己观察到的瞬时信道状态以及相邻车辆间交换的信息，分布式地独立完成资源分配，最优选择子频带以及传输功率。这篇文章综合考虑了V2V链路采用单播或多播两种机制，分别设计了相应的状态、动作空间，以及奖励函数。

3. Resource Allocation in Vehicular Communications using Graph and Deep Reinforcement Learning

这篇文章结合图和DRL设计了车联网中频谱资源的分配机制，研究场景如上图所示。文章优化为题建模为最大化V2I链路和V2V链路的总速率，这篇文章考虑到实际情况中可以获得的信道信息是非常局限的，无法中心式地解决，因此决定将优化问题解耦为信道分配和功率控制两部分，并且只依靠大尺度衰落设计了资源分配方式。

具体而言，频谱分配由基站中心式控制。文章使用图理论对问题建模，对V2I链路采用偶配对算法进行信道分配（最大化V2I链路总速率），对于V2V链路首先采用Max N-Cut算法对V2V车辆进行分簇，使簇内车辆彼此干扰最低，簇内所有V2V链路复用同一信道。然后再使用配对算法为指定V2V簇分配与V2I链路复用的信道（最小化对V2I链路的干扰）。

功率分配由V2V车辆分布式独立完成。考虑到现实情况的可行性，这篇文章对于状态空间设计的非常简单，如下所示：

4. Spectrum Sharing in Vehicular Networks Based on Multi-Agent Reinforcement Learning

这篇文章和前面第二篇文章作者重合，因此研究场景和问题建模非常类似。场景示意图如下所示：

文章假设V2V通信和V2I通信链路共享频谱资源，考虑到实际情况中由于车辆移动性导致信道变化迅速，通过基站集中获取准确的信道信息进行中心式地资源管理分配并不实际。文章提出采用基于多智能体的甚多强化学习算法来解决这个问题。目标是寻找最优的V2V通信子信道分配和功率水平配置方案，在同时实现最大化V2I链路传输速率以及V2V链路成功传输的比率。文章提出的多智能体DRL算法将每一个V2V链路视作agent，各自独立训练自己的DQN网络（基于公共的reward实现协作），训练过程中将其他agent的策略（会随着训练过程发生改变）视作环境的一部分，因此环境是non-stationary的，文章采用基于fingerprint的方法，使agent在训练过程中能够根据其他agent的策略来相应改善提升自己的策略。与之对比，前面第二篇文章是单智能体的算法，通过设定每个agent异步执行动作来规避这个问题。

问题建模比较类似，假设V2I链路集合可表示为M={1,...,M}\mathcal{M}=\left\{1, ..., M\right\}M={1,...,M}，V2V链路集合可表示为K={1,...,K}\mathcal{K}=\left\{1, ..., K\right\}K={1,...,K}，假设V2I链路各自分配了一个正交的子信道，则V2V链路需要复用这MMM个子信道。容易得到V2V和V2I链路各自的SINR以及传输速率表达式，此处略过，优化目标自然是：（1）最大化V2I链路的总速率；（2）最大化V2V链路数据包在时延要求内成功完成发送的概率。

PS: 虽然这篇文章在建模描述的时候与3GPP C-V2X比较接近，但在实际传输建模的时候为了简化没有与mode 4的具体协议标准一致，比如对于时延的处理，这篇文章考虑的是在time budget耗尽前可以一直传输，也就是说可以使用连续的时间子帧，但实际上按照mode 4协议中的semi-persistent scheduling，V2V通信实际上只能周期性地使用不间隔子帧，即在时延范围内（100ms）只能传输一次。
